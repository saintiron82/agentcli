# agentcli v0.6.3

[English](v0.6.3.md) | [한국어](v0.6.3.ko.md)

`ContextSession.fork_many` 추가 — pin 한 컨텍스트의 독립 변형 여러 개를 **병렬**로
실행한다. "큰 입력 1개, 큰 출력 여러 개"용(예: 5만 자 전사록 → 여러 형태의
회의록·보고서를 한 번에).

## 설치

GitHub 우선 릴리즈:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.3"
```

향후 PyPI 릴리즈:

```bash
pip install agentcli-py
```

## 추가

- **`ContextSession.fork_many(prompts, *, concurrency=N, labels=None)`.** pin 한
  컨텍스트의 독립 변형들을 동시 상한과 함께 병렬 실행한다. 각 변형은 전사록을
  자기 세션에 재시드(서로 안 섞임)하고, 결과는 입력 순서대로 반환되며,
  `concurrency` 가 동시 `claude -p` 서브프로세스 수를 제한한다. wall-time 은 합이
  아니라 가장 느린 변형으로 수렴 — 라이브 검증: 5만 자 전사록 → 큰 출력 3개
  (출력 토큰 1.5k–3.2k)를 **~28초**(순차 ~90초+)에, 3개 독립 세션으로. `labels`
  는 기본이 인덱스라 동시성에서 alias 가 결정적·고유하고, `prompts` 와 길이가
  다르면 `ValueError`(prompt 조용한 누락 방지). 큰 출력은 항목별 `fork_stream`
  으로 파일에 흘리거나, lean 을 끄고 에이전트가 직접 파일을 쓰게 한다.
