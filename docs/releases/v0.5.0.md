# agentcli v0.5.0

[English](v0.5.0.md) | [í•śęµ­ě–´](v0.5.0.ko.md)

Internal refactor release. 3 providers (claude / codex / copilot) share a
single streaming skeleton in `LLMProvider`; each provider keeps only its
own JSON-event interpretation. User-facing API and runtime behaviour are
unchanged.

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.5.0"
```

Future PyPI release:

```bash
pip install agentcli
```

## Highlights

- **Common streaming skeleton.** `LLMProvider._run_stream_template(cmd,
  state, ...)` owns subprocess creation, readline loop, idle/wall timeout,
  cleanup, and the `done`/`error` chunk emission. Each provider's
  `stream_async` is now a 30-line wrapper that builds the CLI command,
  seeds a `StreamState`, and delegates.
- **Per-provider event dispatch hook.** `_dispatch_stream_event(evt,
  state)` is the single hook a provider implements to map its CLI's JSON
  event schema to the normalised chunk types (`text` / `thinking` /
  `tool_use` / `tool_result` / `event` / `error` / `done`).
- **Shared async subprocess helper.** `run_subprocess_async(cmd, ...)`
  collapses the proc + `wait_for(communicate, timeout)` + `kill+wait`
  sequence the three `invoke_async` methods used to repeat.
- **Normalisation contract regression tests** (`tests/test_provider_normalization.py`,
  51 cases): declarative attributes, method signatures, `list_models` shape,
  `resolve_model`, binary-missing path, normalised chunk types,
  `build_session_prompt` sharing. Locked before the refactor started ([#7]).

## Fixes

- `CodexProvider._build_cmd` no longer falls back to the literal string
  `"codex"` when the binary is missing. It now returns `None` like
  claude/copilot, and the callers (`invoke` / `invoke_async` /
  `stream_async`) surface `exit_code=127` + `error_type=binary_missing` â€”
  consistent with the 3-provider normalisation contract. The new
  regression tests caught this drift before the refactor started.

## Behaviour changes

- **None for callers.** `LLMClient.chat`, `LLMClient.chat_async`,
  `LLMClient.chat_stream`, `LLMResponse`, and `StreamChunk` keep the same
  signatures, the same field set, and the same per-call behaviour.
- The 3-provider normalisation contract (7 normalised chunk types, session
  is the single source of truth, zero runtime dependencies) is preserved
  and is now explicitly locked by the new regression tests.

## Upgrade notes

- No code changes required.
- If you maintain a private fork that subclassed `ClaudeProvider` /
  `CodexProvider` / `CopilotProvider` and overrode `stream_async`, your
  override still works â€” it shadows the base template. But you can now
  override `_dispatch_stream_event` instead and inherit the common
  skeleton; the relevant base methods are documented hooks.
