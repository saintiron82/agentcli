# agentcli v0.6.3

[English](v0.6.3.md) | [한국어](v0.6.3.ko.md)

Adds `ContextSession.fork_many` — run several independent variations of a
pinned context **in parallel** for the "one big input, several big outputs"
case (e.g. a 50k-char transcript → multiple meeting-minute formats / reports at
once).

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.3"
```

Future PyPI release:

```bash
pip install agentcli-py
```

## Added

- **`ContextSession.fork_many(prompts, *, concurrency=N, labels=None)`.** Runs
  independent variations of the pinned context concurrently with a concurrency
  cap. Each variation re-seeds the transcript in its own session (no
  cross-contamination); results return in input order; `concurrency` bounds the
  simultaneous `claude -p` subprocesses. Wall-time converges to the slowest
  variation rather than the sum — verified live: 3 large outputs (1.5k–3.2k
  output tokens) from a 50k-char transcript in ~28s across 3 independent
  sessions, vs ~90s+ sequential. `labels` default to indices so aliases stay
  deterministic/unique under concurrency; a length mismatch with `prompts`
  raises `ValueError` (no silent prompt drop). For large outputs, `fork_stream`
  each item to a file, or run non-lean so the agent writes the files itself.
