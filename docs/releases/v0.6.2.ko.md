# agentcli v0.6.2

[English](v0.6.2.md) | [한국어](v0.6.2.ko.md)

툴이 필요 없는 단일 completion 을 위한 `ClaudeProvider` **lean 모드**와, 느린
`claude -p` 호출이 어디서 시간을 쓰는지 진단하는 **옵셔널 디버그 계측**을
추가합니다.

## 설치

GitHub 우선 릴리즈:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.2"
```

향후 PyPI 릴리즈:

```bash
pip install agentcli-py
```

## 추가

- **lean 모드 (`ClaudeProvider(lean=True)` / `provider_options={"lean": True}`).**
  툴이 필요 없는 단일 completion(요약·생성·초안)에서, agentcli 가
  `--safe-mode`(CLAUDE.md/skills/plugins/hooks/MCP/custom agents 비활성화) 와
  빌트인 allowlist 인 `--tools`(`""` 면 전부 비활성화, 아니면 명시한
  `allowed_tools`)를 붙인다. 에이전트 하네스를 걷어내, 큰 컨텍스트 completion 이
  MCP 기동 비용을 내지 않고 자율 툴 루프에 빠지지도 않게 한다. lean 에서는
  `mcp_config`/`disallowed_tools` 가 무시된다. 기본 `False` — 기존 동작 불변.

- **옵셔널 디버그 계측 (`debug=True`, `debug_log_path=...`).**
  Claude 의 `--debug` 를 붙이고, 프롬프트가 redact 된 argv 를 로깅하며,
  스트리밍에서는 청크별 타임라인(`+{elapsed}s type evt/name`)을 기록해 툴 루프나
  init 지연을 한눈에 읽을 수 있게 한다. `debug_log_path` 를 주면 JSON Lines
  trace(argv·청크 타임라인·드레인된 stderr·총 소요)가 append 된다. 스트리밍 중
  stderr 를 동시 드레인해, `--debug` 의 대량 출력이 파이프 버퍼를 채워 호출이
  데드락 되는 것을 막는다. `invoke`/`invoke_async`/`stream_async` 와
  `provider_options` 로 전달된다.

- **Claude 토큰 단위 스트리밍 (`partial_messages=True`, 스트리밍 전용).**
  Claude 의 `stream-json` 은 기본적으로 메시지 블록 단위라, 단일 긴 completion 이
  거의 끝에 `text` 청크 하나로 온다("무반응 대기" 체감). `partial_messages` 가
  `--include-partial-messages` 를 붙이고, `_dispatch_stream_event` 가
  `content_block_delta`(`text_delta`/`thinking_delta`) 이벤트를 증분 `text`/
  `thinking` 청크로 매핑한다 — 뒤따르는 전체 `assistant` 블록은 이중 집계
  방지를 위해 건너뛴다. 12초 생성 라이브 A/B: 첫 토큰 ~12초 → ~6초, 청크 1 → 28.

## 수정

- **좀비 손자 프로세스 누적.** CLI 는 자기 자식(MCP 서버·hook·node
  헬퍼)을 띄운다. 기존에는 타임아웃/정리 시 직속 자식만 죽여서 그 손자들이
  계속 살아남았고, stdout 파이프를 물고 있는 손자는 `subprocess.run` 의
  타임아웃 후 정리 단계를 멈추게 할 수도 있었다. 이제 모든 spawn 지점이 새
  세션(프로세스 그룹)으로 띄우고, 타임아웃/취소/조기종료 시 `os.killpg(SIGKILL)`
  로 **그룹 전체**를 정리한다(POSIX; Windows 는 직속 kill 폴백). 새
  `run_subprocess_sync` 가 `ClaudeProvider.invoke` 의 `subprocess.run` 을
  대체해 동일한 그룹 teardown + `stdin=DEVNULL` 을 적용한다. 부모보다 오래 사는
  백그라운드 손자가 타임아웃 시 reap 되는지 결정적 재현 테스트(sync·async·스트리밍)로
  검증.

## 배경

큰 컨텍스트·툴 불필요 작업(예: 약 5만 자 텍스트 → 회의록)의 지배 비용은 출력
토큰 생성(모델에 의존)이지만, 기본값인 `bypassPermissions` + 전체 툴셋 + 설정된
MCP 서버가 기동 비용을 더하고 모델을 수 분짜리 툴 루프로 빠뜨릴 수 있다. lean
모드는 그 표면을 제거하고, debug 모드는 남는 시간을 감사 가능하게 만든다.
