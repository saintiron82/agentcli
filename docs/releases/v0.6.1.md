# agentcli v0.6.1

[English](v0.6.1.md) | [한국어](v0.6.1.ko.md)

Extends call-time MCP support to codex: an embedded codex can now reach external
MCP servers, not just claude.

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.1"
```

Future PyPI release:

```bash
pip install agentcli-py
```

## Added

- **codex `mcp_config` pass-through (AGCLI #154 follow-up).** `CodexProvider`
  `invoke`/`invoke_async`/`stream_async` + `provider_options` accept a per-call
  `mcp_config`. Codex reads MCP servers from `~/.codex/config.toml`, so each
  server is injected per-call as `-c mcp_servers.<name>=<TOML inline table>`
  (a minimal stdlib serializer builds the TOML value). The shape is
  codex-native: `{name: {url, bearer_token_env_var?}}` (HTTP — the token is an
  env-var *name*, not inline headers like claude) or
  `{name: {command, args?, env?}}` (stdio). `mcp_config` is therefore
  provider-shaped (claude vs codex schemas differ).

## Verification

- 405 hermetic tests pass; `build` + `twine check` pass.
- Real codex 0.139.0: an embedded codex spawned by agentcli reached a throwaway
  stdio MCP server via `mcp_config` and performed a **real `mcp_tool_call`**
  (proven by the MCP server recording the call).

## Notes

- In non-interactive `codex exec`, an MCP tool call must clear codex's approval
  gate or it is cancelled (`user cancelled MCP tool call`). The default approval
  config auto-approves; a restrictive `sandbox_mode` + `approval_policy="never"`
  cancels it. Callers tune this via `provider_options` — it is a codex
  operational detail, not an agentcli limitation.
- Zero new runtime dependencies.
- Not yet published to PyPI; install from GitHub at the tag.
