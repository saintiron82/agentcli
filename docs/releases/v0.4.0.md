# agentcli v0.4.0

[English](v0.4.0.md) | [한국어](v0.4.0.ko.md)

`agentcli` is now positioned as a beta Python client layer for embedding the
user's installed Claude Code, Codex, and GitHub Copilot CLI as application
backends.

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.4.0"
```

Future PyPI release:

```bash
pip install agentcli
```

## Highlights

- One `LLMClient` API for Claude Code, Codex, and GitHub Copilot CLI.
- Native CLI sessions remain the source of truth; `agentcli` stores only session
  handles, usage rows, and routing metadata.
- `owner + alias + cwd` identifies long-lived app-level agents.
- Explicit provider/model selection with opt-in fallback.
- Health checks return UI/log-safe output.
- Streaming errors are standardized across providers.
- `AgentProfile.materialize()` writes managed `AGENTS.md` and `CLAUDE.md`
  instructions while protecting user-authored files.

## Status

This is a public beta release. It is suitable for developer integration and
feedback, not a 1.0 stability promise. Each user must install and authenticate
Claude Code, Codex, or GitHub Copilot CLI separately.
