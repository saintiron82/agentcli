# agentcli v0.6.1

[English](v0.6.1.md) | [한국어](v0.6.1.ko.md)

호출 시점 MCP 지원을 codex 까지 확장 — claude 뿐 아니라 임베드된 codex 도 외부
MCP 서버에 닿을 수 있다.

## 설치

GitHub-first 릴리즈:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.1"
```

추후 PyPI 릴리즈:

```bash
pip install agentcli-py
```

## Added

- **codex `mcp_config` 패스스루 (AGCLI #154 follow-up).** `CodexProvider`
  `invoke`/`invoke_async`/`stream_async` + `provider_options` 가 호출 시점
  `mcp_config` 를 받는다. codex 는 MCP 서버를 `~/.codex/config.toml` 에서 읽으므로,
  각 서버를 호출 시점에 `-c mcp_servers.<name>=<TOML inline table>` 로 주입한다
  (stdlib 직렬화기로 TOML 값 생성). 형태는 codex 네이티브 —
  `{name: {url, bearer_token_env_var?}}`(HTTP; 토큰은 inline 헤더가 아니라
  환경변수 *이름*) 또는 `{name: {command, args?, env?}}`(stdio). 따라서
  `mcp_config` 는 provider 별 형태다(claude 와 codex 스키마가 다름).

## 검증

- hermetic 테스트 405건 통과; `build` + `twine check` 통과.
- 실 codex 0.139.0: agentcli 가 띄운 임베드 codex 가 `mcp_config` 로 일회용 stdio
  MCP 서버에 붙어 **실제 `mcp_tool_call`** 수행(서버가 호출을 기록해 증명).

## 비고

- 비대화형 `codex exec` 에서 MCP 도구 호출은 codex 의 승인 게이트를 통과해야
  하며, 그렇지 않으면 취소된다(`user cancelled MCP tool call`). 기본 승인 설정은
  자동 승인하고, 제한적 `sandbox_mode` + `approval_policy="never"` 는 취소한다.
  호출자가 `provider_options` 로 조정하는 codex 운영 디테일이며 agentcli 한계가
  아니다.
- 신규 런타임 의존성 0.
- 아직 PyPI 미배포 — 태그에서 GitHub 설치.
