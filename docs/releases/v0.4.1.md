# agentcli v0.4.1

[English](v0.4.1.md) | [한국어](v0.4.1.ko.md)

Patch release for provider execution and token accounting clarity.

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.4.1"
```

Future PyPI release:

```bash
pip install agentcli
```

## Fixes

- Codex provider invocation now resolves the actual CLI binary path before
  running `codex exec`, matching the health-check path. This fixes Windows
  installs where `codex.CMD` is discoverable through `PATHEXT` but bare
  `subprocess.run(["codex", ...])` fails.
- Token usage now separates provider CLI reported prompt tokens from the prompt
  payload estimate that agentcli sent to the CLI.

## Token Reporting

`TokenUsage` now includes:

- `payload_prompt_tokens`: lightweight estimate of the prompt string agentcli
  passed to the provider CLI.
- `prompt_tokens_reliable`: whether `prompt_tokens` is safe for portable
  prompt-size comparisons.
- `prompt_tokens_source`: the source label for provider-reported prompt tokens.

`get_token_stats()` also includes `total_payload_prompt` and
`prompt_tokens_unreliable_calls`.
