# agentcli v0.4.3

[English](v0.4.3.md) | [한국어](v0.4.3.ko.md)

Claude provider의 `-p` 모드와 SQLite 세션 매니저 충돌을 보정한 패치 릴리즈입니다.

## 설치

GitHub-first 릴리즈:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.4.3"
```

향후 PyPI 릴리즈:

```bash
pip install agentcli
```

## 수정

- Claude provider가 더 이상 `-p` (print/single-shot) 호출에 `--resume
  <session_id>`를 붙이지 않습니다. 두 모드는 구조적으로 호환되지 않습니다 —
  `-p`는 stateless라 복원할 인터랙티브 세션이 없고, `--resume`은 인터랙티브
  모드 전용 플래그입니다. Windows 환경에서 이 조합이 5분 hang 후
  `wall_timeout` 실패를 유발했습니다 ([#4]).
- `ClaudeProvider.supports_sessions`가 `False`로 변경되었습니다. 라이브러리는
  더 이상 `SQLiteStore` / `MemoryStore`에 `session_id:claude` 메타데이터를
  저장하거나 재생하지 않습니다. 따라서 stale Claude session_id가 다음
  호출로 흘러갈 수 없습니다.

## 동작 변경

- `LLMClient.chat(..., provider="claude")`은 더 이상 호출 간 세션 연속성을
  유지하지 않습니다. `claude -p`는 원래 single-shot이었고, 이번 릴리즈는
  그 사실을 코드 레벨에서 명시화한 것입니다.
- v0.4.3 이후 호출에 대해 `LLMClient.get_alias_status(...)`는 `claude`를
  `session_providers`에 포함시키지 않습니다. 이전 버전에서 SQLite에 저장된
  `session_id:claude` 메타데이터는 남아있더라도 사용되지 않습니다 (무해).
- 매 호출마다 `--session-id <uuid>`는 계속 발급되어, usage 로그가 호출별
  안정적 식별자를 가질 수 있습니다.

## 업그레이드 노트

- 앱 코드 변경은 필요 없습니다.
- 이번 릴리즈 전에 우회용으로 `client.clear_session_metadata(provider="claude")`
  를 호출하고 계셨다면 그 호출은 제거해도 됩니다.
- 3-프로바이더 정규화 계약은 Codex와 Copilot에 대해 그대로 유지됩니다 —
  구조적으로 `-p` 단일 모드인 것은 Claude뿐이었습니다.
