# agentcli v0.4.3

[English](v0.4.3.md) | [한국어](v0.4.3.ko.md)

Patch release for the Claude provider's `-p` mode / SQLite session conflict.

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.4.3"
```

Future PyPI release:

```bash
pip install agentcli
```

## Fixes

- Claude provider no longer attaches `--resume <session_id>` to the
  `-p` (print/single-shot) invocation. The two are structurally
  incompatible — `-p` is stateless and has no interactive session to
  resume, while `--resume` is an interactive-mode flag. On Windows the
  combination caused a 5-minute hang followed by a `wall_timeout` ([#4]).
- `ClaudeProvider.supports_sessions` is now `False`. The library no
  longer stores or replays `session_id:claude` metadata in
  `SQLiteStore` / `MemoryStore`, so a stale Claude session id can never
  reach a subsequent call.

## Behavior changes

- `LLMClient.chat(..., provider="claude")` no longer carries session
  continuity between calls. `claude -p` was always single-shot — this
  release just stops pretending otherwise.
- `LLMClient.get_alias_status(...)` will not list `claude` in
  `session_providers` for calls made under v0.4.3+. Any stale
  `session_id:claude` metadata left over from earlier versions in your
  SQLite store is harmless but unused.
- A fresh `--session-id <uuid>` is still passed on every call so usage
  rows have a stable per-call identifier for audit.

## Upgrade notes

- No code changes are required.
- If your app called `client.clear_session_metadata(provider="claude")`
  before this release as a workaround, you can drop that call.
- The 3-provider normalization contract is unchanged for Codex and
  Copilot — only Claude was structurally `-p`-only.
