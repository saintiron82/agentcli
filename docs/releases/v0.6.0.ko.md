# agentcli v0.6.0

[English](v0.6.0.md) | [한국어](v0.6.0.ko.md)

네 번째 provider(Kiro, ACP 기반)와 호출 시점 `provider_options` 추가 — 임베드된
에이전트가 외부 MCP 서버에 닿고, 호출마다 읽기↔행위를 전환할 수 있다.

## 설치

GitHub-first 릴리즈:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.0"
```

추후 PyPI 릴리즈:

```bash
pip install agentcli-py
```

## Added

- **KiroProvider** — 네 번째 provider. `kiro-cli acp`(ACP, 줄 단위 JSON-RPC 2.0
  over stdio)를 호출당 1회 one-shot turn 으로 감싸 세션 연속성·타입드
  스트리밍·토큰 통계를 제공. 외부 `LLMProvider` 계약과 7개 정규화 청크 타입은
  기존 3종과 동일. 런타임 의존성 0.
- **호출 시점 `provider_options` — MCP & 권한 오버라이드 (AGCLI #154).**
  `LLMClient.chat`/`chat_async`/`chat_stream` 가 `provider_options: dict | None`
  를 받는다. 키는 그 키를 받는 provider 에게만 전달되어, 한 provider 전용
  옵션(예: claude `mcp_config`)은 다른 provider 로 fallback 돼도 안전하게
  무시된다. `ClaudeProvider` 가 `--mcp-config`(+`--strict-mcp-config`)를 방출해
  임베드된 claude 가 외부 MCP 서버(예: Pair)에 닿을 수 있고,
  `permission_mode`/`allowed_tools` 만 바꿔 같은 세션에서 읽기→행위를 오간다.
  `CodexProvider` 는 `sandbox_mode`/`approval_policy` 오버라이드(행위 턴은
  `new_session=True` 와 병행 — resume 는 `-s` 무시).

## Fixed

- **KiroProvider `session/new`/`session/load` 에 ACP 필수 `mcpServers` 필드
  누락** — 실 `kiro-cli` 대상 매 턴 stall(idle timeout)의 원인. `{"cwd": ...}`
  만 보내 kiro 가 응답하지 않았다. `"mcpServers": []` 추가로 수정. raw ACP
  spike(`kiro-cli login` 계정 인증으로 실행)로 나머지 필드 매핑
  (`sessionUpdate`/`content.text`/`stopReason`)이 `kiro-cli` 2.8.1 과 일치함을
  실측 확인.

## 검증

- hermetic 테스트 400건 통과; `build` + `twine check` 통과.
- 실 `claude` CLI: `--mcp-config '{"mcpServers": {}}'` + `--strict-mcp-config`
  수용 확인. 임베드 claude 가 **외부 MCP 실제 도구 호출 1건** 수행(일회용 MCP
  서버가 `tools/call` 을 기록해 증명).
