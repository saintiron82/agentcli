# agentcli v0.6.4

[English](v0.6.4.md) | [한국어](v0.6.4.ko.md)

`debug` 를 1급·크로스 프로바이더 기능으로 승격한다. 이제 debug 는 claude 뿐
아니라 **codex·copilot** 에서도, **스트리밍·논스트리밍** 양쪽 호출 경로 모두에서
동작하므로 `supports(provider, "debug")` 가 모든 호출에 대해 정직하다.

## 설치

GitHub 우선 릴리즈:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.4"
```

향후 PyPI 릴리즈:

```bash
pip install agentcli-py
```

## 추가

- **debug 계측, 크로스 프로바이더, 양쪽 경로.** `stream_async` 는 이미
  `debug` / `debug_log_path` 를 받았고, 이제 `invoke` / `invoke_async` 도 모든
  프로바이더에서 받는다. 스트리밍은 공유 `_run_stream_template` 로 위임(청크별
  타임라인 + 동시 stderr drain), 논스트리밍은 공유 `emit_invoke_debug` 헬퍼로
  argv/rc/latency/stderr 트레이스를 방출한다(claude 의 `_emit_invoke_debug` 도
  이제 여기로 위임). 논스트리밍 codex/copilot 호출에서 조용히 무시되던 동작이
  사라졌다. kiro 는 범위 외(stream-json 템플릿이 아닌 ACP 전송).
- **트레이스 견고성.** 각 트레이스 레코드는 `schema`(포맷 버전, 현재 `1`)와 고유
  `call_id` 를 갖는다. 병렬 `fork_many` 실행에서 섞인 트레이스를 그룹핑할 수 있고
  소비자가 포맷별로 분기할 수 있다. `write_debug_trace` 는 락으로 쓰기를
  직렬화하여 여러 스레드의 동시 append 가 한 줄을 손상시키지 못하게 한다.
- **`debug` 가 정식 capability.** `ProviderCapabilities.debug` 와
  `capability_matrix()` 행이 어떤 프로바이더에 debug 계측이 있는지 보여준다
  (claude/codex/copilot ✓, kiro ✗). `supports(provider, "debug")` 가 스트리밍·
  논스트리밍 양쪽 호출에 대해 정확히 이를 반영한다.
