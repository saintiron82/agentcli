# agentcli v0.5.1

[English](v0.5.1.md) | [한국어](v0.5.1.ko.md)

Real-world embedding release: Claude session resume restored on
macOS/Linux, explicit history modes, and provider-layer hardening.

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.5.1"
```

Future PyPI release:

```bash
pip install agentcli
```

## Added

- **Claude native session resume (macOS/Linux).**
  `ClaudeProvider.supports_sessions` is `True` on POSIX: the first call
  mints `--session-id`, later calls on the same conversation pass
  `-p --resume <sid>` (verified against Claude Code 2.1.x — the resumed
  session keeps the same ID). Windows stays stateless (issue #4 hang
  workaround).
- Stale-session auto-recovery: when a stored Claude session id no longer
  exists ("No conversation found"), the call retries once with a fresh
  session — in `invoke`, `invoke_async`, and `stream_async`.
- `LLMProvider.stores_history` contract: the three CLI providers never
  persist conversation content or re-inject prior turns, even in
  non-session mode (Windows Claude). Custom providers default to `True`.
- **Explicit history modes per call**:
  1. CLI-native session (default) — same `owner` + `alias` resumes the
     provider's own session.
  2. Host-injected context — `inject_context` now reaches session
     providers too (previously a silent no-op for all three CLIs);
     injected messages are serialized into the prompt as a labeled
     `Context` block.
  3. No history — new `new_session=True` runs one call in a fresh CLI
     session; the alias then tracks the new session.
- Same-conversation concurrency protection (in-process): concurrent
  calls on one conversation serialize (sync `threading.Lock`, async
  per-loop `asyncio.Lock`) and re-read the session id under the lock,
  so parallel calls no longer fork/overwrite CLI sessions.
- Example: `examples/05_review_system.py` — a two-agent review pipeline
  (merge-gate review → repairman execution-card drafts) driven entirely
  through agentcli with natively loaded Agent Skills.

## Fixed

- Copilot streaming now emits normalized `error` chunks for `error`,
  `assistant.error`, `session.error`, and `result.exitCode != 0` events
  (previously disguised as `event` chunks, so hosts could not detect
  failures mid-stream).
- Batch JSONL parsers (all three providers) and the shared stream
  template no longer raise `AttributeError` into the host on valid JSON
  lines that are not objects.
- `run_subprocess_async` now cleans up the subprocess on task
  cancellation (`CancelledError`) — the invoke-path twin of the v0.5.0
  streaming cleanup fix.
- Codex positional arguments (prompt, resume session id) are passed
  after a `--` separator, so untrusted input starting with `-` can no
  longer be parsed as CLI flags.

## Behavior changes

- **Breaking on macOS/Linux**: Claude calls on the same
  `owner` + `alias` now carry session continuity (v0.4.3–v0.5.0 were
  fully stateless). Pass `new_session=True` or use
  `clear_session_metadata()` for the old per-call-fresh behavior.
  Windows behavior is unchanged.
- E2E-verified through the library against Claude Code 2.1.x: session
  continuity, `new_session` isolation, `inject_context` delivery, and
  native pickup of project `CLAUDE.md`, Agent Skills
  (`.claude/skills/`), and custom subagents (`.claude/agents/`) from
  `cwd`.

## Compatibility

- Zero runtime dependencies, unchanged.
- Streaming chunk contract unchanged (7 types) — Copilot now actually
  honors the `error` type mid-stream.
- 349 tests passing.
