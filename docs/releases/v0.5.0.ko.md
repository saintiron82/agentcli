# agentcli v0.5.0

[English](v0.5.0.md) | [한국어](v0.5.0.ko.md)

내부 리팩토링 릴리즈. 세 provider (claude / codex / copilot) 가 `LLMProvider`
의 공통 스트리밍 골격을 공유하고, 각 provider 는 자기 CLI 의 JSON event
해석만 책임집니다. 사용자 API 와 동작은 변경 없습니다.

## 설치

GitHub-first 릴리즈:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.5.0"
```

향후 PyPI 릴리즈:

```bash
pip install agentcli
```

## 주요 변경

- **공통 스트리밍 골격.** `LLMProvider._run_stream_template(cmd, state, ...)`
  가 subprocess 생성, readline 루프, idle/wall timeout, cleanup, `done`/`error`
  chunk 방출을 모두 소유합니다. 각 provider 의 `stream_async` 는 CLI 명령을
  조립하고 `StreamState` 를 초기화한 뒤 위임하는 30줄짜리 wrapper 가 됐습니다.
- **Provider 별 event dispatch hook.** `_dispatch_stream_event(evt, state)`
  하나만 구현하면 자기 CLI 의 JSON event 스키마를 정규화된 chunk
  타입 (`text` / `thinking` / `tool_use` / `tool_result` / `event` /
  `error` / `done`) 으로 매핑합니다.
- **공통 async subprocess helper.** `run_subprocess_async(cmd, ...)` 가
  3 provider `invoke_async` 가 반복하던 proc + `wait_for(communicate,
  timeout)` + `kill+wait` 시퀀스를 한 곳으로 모았습니다.
- **정규화 계약 회귀 테스트** (`tests/test_provider_normalization.py`,
  51 케이스): declarative attribute / 메서드 시그니처 / `list_models`
  shape / `resolve_model` / 바이너리 부재 path / 정규화 chunk 타입 집합 /
  `build_session_prompt` 공유. 리팩토링 시작 전에 invariant 를 잠갔습니다 ([#7]).

## 내부 수치

| 파일 | Before | After | Δ |
|------|------:|------:|------:|
| `providers/claude.py` | 487 | 391 | **-96** |
| `providers/codex.py` | 600 | 512 | **-88** |
| `providers/copilot.py` | 565 | 472 | **-93** |
| `providers/base.py` | 248 | 410 | +162 (helper + hook) |
| **providers/ 합계** | 1,894 | 1,845 | **순 -49, 중복 약 280줄을 base 한 곳으로 흡수** |

4번째 provider 를 추가할 때 이전엔 ~150줄짜리 `stream_async` 를 통째로
복사해야 했습니다. 이번 릴리즈 이후엔 hook 4개 (`_build_cmd`,
`_dispatch_stream_event`, `invoke`, `health_check`) 만 구현하면 됩니다.

## 수정

- `CodexProvider._build_cmd` 가 바이너리 부재 시 `"codex"` 문자열로 fallback
  하던 정규화 계약 위반을 수정했습니다. claude/copilot 와 동일하게 `None`
  반환 → 호출자 (`invoke` / `invoke_async` / `stream_async`) 가 즉시
  `exit_code=127`, `error_type=binary_missing` 으로 실패합니다. 새 회귀
  테스트가 리팩토링 시작 단계에서 잡았습니다.

## 동작 변경

- **호출자 입장에서 변경 없음.** `LLMClient.chat`, `LLMClient.chat_async`,
  `LLMClient.chat_stream`, `LLMResponse`, `StreamChunk` 의 시그니처·필드·
  호출별 동작 모두 그대로입니다.
- 3-provider 정규화 계약 (정규화된 chunk 타입 7종, 세션은 CLI 단일 진실,
  런타임 의존성 0) 은 보존되며, 이번 릴리즈부터 새 회귀 테스트로
  명시적으로 잠겨 있습니다.

## 업그레이드 노트

- 코드 변경은 필요 없습니다.
- 만약 `ClaudeProvider` / `CodexProvider` / `CopilotProvider` 를 private
  fork 에서 서브클래싱해 `stream_async` 를 override 하셨다면, 그 override
  는 그대로 동작합니다 (base template 을 가립니다). 다만 이번부터는
  `_dispatch_stream_event` 만 override 하고 공통 골격을 상속받는 패턴이
  권장됩니다. 관련 base 메서드는 hook 으로 문서화돼 있습니다.
