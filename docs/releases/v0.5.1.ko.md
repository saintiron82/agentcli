# agentcli v0.5.1

[English](v0.5.1.md) | [한국어](v0.5.1.ko.md)

실사용 임베딩 릴리즈: macOS/Linux에서 Claude 세션 resume 복구, 히스토리
모드 명시화, provider 계층 견고화.

## 설치

GitHub 우선 릴리즈:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.5.1"
```

향후 PyPI 릴리즈:

```bash
pip install agentcli
```

## 추가

- **Claude 네이티브 세션 resume (macOS/Linux).**
  POSIX에서 `ClaudeProvider.supports_sessions = True` — 첫 호출은
  `--session-id`를 발급하고, 같은 conversation의 이후 호출은
  `-p --resume <sid>`로 재개합니다 (Claude Code 2.1.x 실검증 — resume
  후에도 동일 ID 유지). Windows는 issue #4 hang 회피를 위해 무상태
  유지.
- 만료 세션 자동 복구: 저장된 Claude session id의 네이티브 세션이
  사라진 경우("No conversation found") 새 세션으로 1회 재시도 —
  `invoke` / `invoke_async` / `stream_async` 모두.
- `LLMProvider.stores_history` 계약: 3개 CLI provider는 비세션
  모드(Windows Claude)에서도 대화 내용을 저장하거나 이전 턴을
  재주입하지 않습니다. custom provider 기본값은 `True`.
- **호출 단위 히스토리 3-모드**:
  1. CLI 네이티브 세션(기본) — 같은 `owner` + `alias`로 provider 자체
     세션 resume.
  2. 호스트 주입 — `inject_context`가 세션 provider에도 적용
     (이전에는 3-CLI에서 무동작). 주입 메시지는 라벨된 `Context`
     블록으로 프롬프트에 직렬화.
  3. 미사용 — 신규 `new_session=True`: 이 호출만 새 CLI 세션에서
     시작하고, 이후 alias는 새 세션을 추적.
- 같은 conversation 동시 호출 보호 (in-process): 동시 호출은
  직렬화되고(sync `threading.Lock`, async 루프별 `asyncio.Lock`)
  잠금 후 session id를 재조회하므로, 병렬 호출이 CLI 세션을
  분기/덮어쓰지 않습니다.
- 예제: `examples/05_review_system.py` — merge-gate 리뷰 → repairman
  실행 카드 초안의 2-에이전트 리뷰 파이프라인. agentcli와 네이티브
  로드된 Agent Skills만으로 구동.

## 수정

- Copilot 스트리밍이 `error` / `assistant.error` / `session.error` /
  `result.exitCode != 0` 이벤트를 정규화된 `error` 청크로 방출
  (이전에는 `event`로 위장되어 호스트가 스트림 중 실패를 감지할 수
  없었음).
- 배치 JSONL 파서(3-provider)와 공통 스트림 템플릿이 "유효한 JSON
  이지만 객체가 아닌" 라인에서 `AttributeError`를 호스트로 전파하던
  문제 수정.
- `run_subprocess_async`가 task 취소(`CancelledError`) 시에도
  subprocess를 정리 — v0.5.0 스트리밍 정리 수정의 invoke 경로 대응.
- Codex 위치 인자(prompt, resume session id)를 `--` 구분자 뒤로 전달
  — `-`로 시작하는 untrusted 입력의 플래그 주입 차단.

## 동작 변경

- **macOS/Linux breaking change**: 같은 `owner` + `alias`의 Claude
  호출이 이제 세션 연속성을 가집니다 (v0.4.3~v0.5.0은 완전 무상태).
  이전처럼 매 호출 새로 시작하려면 `new_session=True` 또는
  `clear_session_metadata()`를 사용하세요. Windows 동작은 변경 없음.
- 라이브러리 경유 E2E 검증 (Claude Code 2.1.x): 세션 연속성,
  `new_session` 격리, `inject_context` 전달, 그리고 `cwd`의 프로젝트
  `CLAUDE.md` / Agent Skills(`.claude/skills/`) / 커스텀
  서브에이전트(`.claude/agents/`) 네이티브 로드.
