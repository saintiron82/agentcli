# agentcli v0.6.0

[English](v0.6.0.md) | [한국어](v0.6.0.ko.md)

Adds a fourth provider (Kiro, via ACP) and call-time `provider_options`, so an
embedded agent can reach external MCP servers and switch read↔act per call.

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.0"
```

Future PyPI release:

```bash
pip install agentcli-py
```

## Added

- **KiroProvider** — a fourth provider that wraps `kiro-cli acp` (ACP,
  line-delimited JSON-RPC 2.0 over stdio) as a one-shot turn per call: session
  continuity + typed streaming + token usage, with the same external
  `LLMProvider` contract and 7 normalized chunk types as the other three. Zero
  runtime dependencies.
- **Call-time `provider_options` — MCP & permission overrides (AGCLI #154).**
  `LLMClient.chat`/`chat_async`/`chat_stream` accept `provider_options: dict |
  None`; keys are forwarded only to providers whose call accepts them, so one
  provider's option (e.g. claude `mcp_config`) is harmlessly ignored on fallback
  to another. `ClaudeProvider` emits `--mcp-config` (+ `--strict-mcp-config`) so
  an embedded claude can reach external MCP servers (e.g. Pair) and switch
  read→act on the same session by changing only `permission_mode` /
  `allowed_tools`. `CodexProvider` accepts `sandbox_mode` / `approval_policy`
  (an acting turn pairs with `new_session=True`, since resume ignores `-s`).

## Fixed

- **KiroProvider `session/new` / `session/load` were missing the ACP-required
  `mcpServers` field**, which stalled every turn against the real `kiro-cli`
  until the idle timeout. Now sends `"mcpServers": []`. A raw ACP spike (run via
  `kiro-cli login` account auth) confirmed the rest of the field mapping
  (`sessionUpdate` / `content.text` / `stopReason`) is correct against
  `kiro-cli` 2.8.1.

## Verification

- 400 hermetic tests pass; `build` + `twine check` pass.
- Real `claude` CLI: `--mcp-config '{"mcpServers": {}}'` + `--strict-mcp-config`
  accepted; an embedded claude performed a **real external-MCP tool call**
  (proven by a throwaway MCP server recording the `tools/call`).
