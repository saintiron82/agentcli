# agentcli v0.4.1

[English](v0.4.1.md) | [한국어](v0.4.1.ko.md)

provider 실행과 토큰 집계 의미를 정리한 패치 릴리즈입니다.

## 설치

GitHub-first 릴리즈:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.4.1"
```

향후 PyPI 릴리즈:

```bash
pip install agentcli
```

## 수정

- Codex provider가 `codex exec`를 실행하기 전에 실제 CLI binary 경로를
  resolve합니다. health check와 같은 경로를 쓰므로 Windows의 `codex.CMD`
  설치에서도 bare `subprocess.run(["codex", ...])` 실패를 피합니다.
- provider CLI가 보고한 prompt token과 agentcli가 CLI에 넘긴 prompt payload
  추정치를 분리했습니다.

## 토큰 리포팅

`TokenUsage`에 다음 필드가 추가되었습니다.

- `payload_prompt_tokens`: agentcli가 provider CLI에 넘긴 prompt 문자열의
  가벼운 추정 토큰 수.
- `prompt_tokens_reliable`: `prompt_tokens`를 provider 간 prompt 크기 비교에
  써도 되는지 여부.
- `prompt_tokens_source`: provider-reported prompt token의 출처 라벨.

`get_token_stats()`도 `total_payload_prompt`와
`prompt_tokens_unreliable_calls`를 반환합니다.
