# agentcli v0.6.4

[English](v0.6.4.md) | [한국어](v0.6.4.ko.md)

Promotes `debug` to a first-class, cross-provider capability. Debug now works
for **codex and copilot** too — not just claude — on **both** the streaming and
non-streaming call paths, so `supports(provider, "debug")` is honest for every
call.

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.4"
```

Future PyPI release:

```bash
pip install agentcli-py
```

## Added

- **Debug instrumentation, cross-provider, both paths.** `stream_async` already
  accepted `debug` / `debug_log_path`; now `invoke` / `invoke_async` do too, for
  every provider. Streaming delegates to the shared `_run_stream_template`
  (per-chunk timeline + concurrent stderr drain); non-streaming emits an
  argv/rc/latency/stderr trace via a shared `emit_invoke_debug` helper (claude's
  `_emit_invoke_debug` now delegates to it). No more silent no-op on a
  non-streaming codex/copilot call. kiro stays out of scope (ACP transport, not
  the stream-json template).
- **Trace robustness.** Every trace record carries a `schema` (format version,
  currently `1`) and a unique `call_id`, so interleaved traces from parallel
  `fork_many` runs are groupable and consumers can branch on format.
  `write_debug_trace` serializes writes with a lock so concurrent appends from
  multiple threads can't corrupt a line.
- **`debug` is a declared capability.** `ProviderCapabilities.debug` and the
  `capability_matrix()` row show which providers have debug instrumentation
  (claude/codex/copilot ✓, kiro ✗); `supports(provider, "debug")` reflects it,
  accurately, for both streaming and non-streaming calls.
