# agentcli v0.6.2

[English](v0.6.2.md) | [한국어](v0.6.2.ko.md)

Adds a `ClaudeProvider` **lean mode** for tool-free single-shot completions and
**optional debug instrumentation** to diagnose where a slow `claude -p` call
spends its time.

## Install

GitHub-first release:

```bash
pip install "agentcli @ git+https://github.com/saintiron82/agentcli.git@v0.6.2"
```

Future PyPI release:

```bash
pip install agentcli-py
```

## Added

- **Lean mode (`ClaudeProvider(lean=True)` / `provider_options={"lean": True}`).**
  For a single completion that needs no tools (summarize, generate, draft),
  agentcli appends `--safe-mode` (disables CLAUDE.md/skills/plugins/hooks/MCP/
  custom agents) and `--tools` as a built-in allowlist (`""` to disable all
  tools, or the explicit `allowed_tools`). This strips the agent harness so a
  large-context completion does not pay for MCP startup and cannot fall into an
  autonomous tool loop. `mcp_config`/`disallowed_tools` are ignored in lean
  mode. Default `False` — existing behavior is unchanged.

- **Optional debug instrumentation (`debug=True`, `debug_log_path=...`).**
  Appends Claude's `--debug`, logs the prompt-redacted argv, and — for
  streaming — records a per-chunk timeline (`+{elapsed}s type evt/name`) so a
  tool loop or an init stall is legible at a glance. When `debug_log_path` is
  set, a JSON-Lines trace (argv, chunk timeline, drained stderr, elapsed) is
  appended. During streaming, stderr is drained concurrently so Claude's
  `--debug` output cannot fill the pipe buffer and deadlock the call. Threaded
  through `invoke`/`invoke_async`/`stream_async` and `provider_options`.

## Fixed

- **Orphaned grandchild processes ("zombie" accumulation).** A CLI spawns its
  own children (MCP servers, hooks, node helpers). Previously, killing only the
  direct child on timeout/cleanup left those grandchildren alive, and a
  grandchild holding the stdout pipe could wedge `subprocess.run`'s post-timeout
  cleanup. All spawn sites now start a new session (process group) and tear down
  the **whole group** via `os.killpg(SIGKILL)` (POSIX; Windows falls back to a
  direct kill). A new `run_subprocess_sync` replaces `subprocess.run` in
  `ClaudeProvider.invoke` for the same group teardown plus `stdin=DEVNULL`.
  Confirmed with a deterministic repro: a backgrounded grandchild that outlives
  its parent is reaped on timeout (sync and async).

## Why

For a large-context, tool-free task (e.g. ~50k-char text → meeting minutes),
the dominant cost is output-token generation (model-bound), but the default
`bypassPermissions` + full toolset + configured MCP servers add startup cost
and can divert the model into a tool loop that runs for many minutes. Lean mode
removes that surface; debug mode makes the remaining time auditable.

## Verification

- 411 hermetic tests pass (lean/debug command-building + streaming-trace unit
  tests added); `build` + `twine check` pass.
- Real Claude Code 2.1.x: lean + debug streaming verified end-to-end — argv is
  redacted, the chunk timeline is recorded with timestamps, and concurrent
  stderr draining holds (no deadlock).

## Notes

- `lean`/`debug` map to Claude Code flags, so they are Claude-specific. The
  client's `provider_options` filtering means Codex/Copilot safely ignore them;
  equivalents for those providers are not implemented.
- Zero new runtime dependencies.
- Not yet published to PyPI; install from GitHub at the tag.
